# 模板构造教程

本文件介绍如何从一个空白文件夹建立起一个可以使用的模板工程。

## 一、库文件获取

在创建具体的工程文件夹之前，需要获取 `对应MCU` 的库文件。

> 当前， ST 已经停止了对于标准库的维护。因此本文将完全以 HAL 库作为唯一的 `硬件抽象层库` 。

### 1.1 CMSIS 库

全称：`Common Microcontroller Software Interface Standard` - 通用微控制器软件接口标准。

CMSIS 是 ARM 官方定义的一套 Cortex-M 微控制器软件标准。其核心的作用是 规范并实现一套 `统一的` 标准：

- 内核寄存器和中断控制的访问
- 启动文件、异常向量表、系统时钟初始化框架
- 编译器/内核相关的接口
- 包括 DSP、RTOS 抽象接口等

CMSIS 提供底层统一规范和内核/设备基础定义，而 ST 公司提供的 HAL 库就是使用这些定义去实现 STM32 外设驱动。

#### 1.1.1 CMSIS 库的获取

```URL
https://github.com/ARM-software
```

上方的地址是 ARM 公司在 github 的官方组织地址，其中绝大多数的情况下，需要使用的是 CMSIS_5 这个仓库里的内容：

```URL
https://github.com/ARM-software/CMSIS_5
```

进入以上网址并找该仓库的最后一版 Releases ，并下载其源码。

> 对于嵌入式单片机开发来说，CMSIS_5 的库完全足够，同时 CMSIS_6 是 ARM 想构建的新软件生态，在单片机领域生态里几乎还没普及。


### 1.2 HAL 库

全称：`Hardware Abstraction Layer` - 硬件抽象层

ST 为其出售的芯片提供了一系列的 HAL 库。HAL 库的核心作用是提供一阵套 API，为应用层开发提供 `外围设备` 与 `外部设备` 驱动，便于应用开发。

#### 1.2.1 HAL 库的获取

```URL
https://github.com/STMicroelectronics
```
上方的地址是 ARM 公司在 github 的官方组织地址，ST 封装的所有 HAL 库均可以在其中找到。

对于 `STM32F429IGT6` 芯片来说，该芯片属于 `STM32F4` 系列。所以应使用如下 `F4` 系列 HAL 库：

```URL
https://github.com/STMicroelectronics/stm32f4xx-hal-driver
```

以及 ST 公司为 CMSIS 库对接的部分核心文件：

```URL
https://github.com/STMicroelectronics/cmsis-device-f4
```

克隆这两个仓库到本地。

> 对于其他系列的芯片，请自行寻找对应的库文件。
